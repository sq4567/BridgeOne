# 범용 디버깅 & 복잡한 문제 해결

복잡한 문제 분석, 근본 원인 조사, 다중 컴포넌트 간 상호작용 문제를 체계적으로 해결합니다.

이 커맨드는 다음 상황에 유용합니다:
- **간헐적 오류**: 재현이 어렵고 원인 파악이 힘든 문제
- **다중 컴포넌트 문제**: 여러 시스템이 연관된 복잡한 실패
- **성능 문제**: 원인이 명확하지 않은 느린 응답
- **예상 밖의 동작**: 잘못된 결과나 비정상 상황
- **아키텍처 이슈**: 컴포넌트 간 설계 또는 통합 문제

---

## 사용 방법

**형식**:
```
/debug "문제 설명" [--scope 범위] [--context 추가 정보]
```

**예시**:
```
/debug "USB 연결이 간헐적으로 끊어짐" --scope esp32,android,usb --context "주로 5분 이상 유휴 상태일 때 발생"

/debug "키보드 입력이 가끔 실패함" --scope hid,protocol --context "특정 키는 항상 실패, 로그에 오류 없음"

/debug "Android 앱 메모리 누수" --scope android,memory --context "30분 사용 후 점점 느려짐, 최대 500MB까지"
```

---

## 분석 프로세스

1️⃣ **문제 정의**: 증상, 환경, 재현 조건 명확화
2️⃣ **범위 결정**: 영향받는 컴포넌트/시스템 식별
3️⃣ **근본 원인 분석**: 공식 문서 참고하여 잠재 원인 파악
4️⃣ **가설 테스트**: 단계별 검증 방법 제시
5️⃣ **해결책**: 실행 가능한 수정 사항 제시
6️⃣ **예방**: 유사 문제 방지 방법 안내

---

## 정보 제공 팁

문제 설명 시 다음 정보를 포함하면 분석이 더 정확합니다:

**필수 정보**:
- ✅ 문제 증상 (무엇이 잘못되었는가)
- ✅ 재현 방법 (언제, 어떻게 발생하는가)
- ✅ 영향 받는 컴포넌트 (Android, ESP32, PC 등)

**추가 정보** (있으면 포함):
- 에러 메시지 또는 로그
- 최근 코드 변경 사항
- 이전에 시도한 해결책
- 환경 설정 (OS, 버전, 드라이버 등)
- 발생 빈도 (항상/간헐적/특정 조건)

## 예상 출력

systematic-problem-solver를 통해 다음을 받게 됩니다:

- 📋 **문제 분석 보고서**: 증상 요약 및 영향 범위
- 🔍 **근본 원인 후보**: 가능한 원인들과 가능성 평가
- 📚 **공식 문서 참조**: Context7를 통한 관련 문서 링크
- 🧪 **검증 방법**: 각 가설을 테스트하는 방법
- ✅ **해결책**: 단계별 수정 방법
- 🛡️ **예방 방안**: 향후 유사 문제 방지 방법
